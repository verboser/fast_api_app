В этом файле представлено руковоство по запуску и тестированию данного приложения.

Задание было выполнено в соотвествии с необходимыми критериями.
Так как было принято решение сделать два варианта запуска (через Docker Compose и через Swarm), прошу обратить внимание, что есть два режима в приложении:

**ВАЖНО**: Перед запуском и тестом запустите docker у себя на ПК или ноутбуке.

**Сервис запускается по этому адрессу:** http://127.0.0.1:8501

### Режим 1: Docker Compose

Запуск:

Откройте терминал в корневой папке проекта (там, где лежит docker-compose.yml)

Введите команду для сборки образов и фонового запуска:
    ```
    docker-compose up -d --build
    ```
Для тестирования используйте следующие команды: 

- Для тестирования отказоустойчивости базы данных:
    ```
    docker exec <container_id> sh -c "kill -9 1" 
    ```
  Используется именно этот вариант поломки, потому что докер воспринимает команду `docker kill` на бызу данных как право администратора


- Для тестирования отказоустойчивости самого сервиса:
    ```
    docker kill <container_id>
    ```
  Тут всё должно пройти без сучка и задоринки

Остановка и очистка:
    ```
    docker-compose down
    ```

### Режим 2: Docker Swarm 

Важно перед запуском: Убедитесь, что обычный сompose остановлен командой docker-compose down, иначе Swarm выдаст ошибку занятых портов

Инициализируйте режим кластера (нужно сделать только один раз):

    docker swarm init

Соберите локальные образы (Swarm не умеет собирать код на лету с помощью флага --build):

    docker-compose build

Запустите стек сервисов под управлением оркестратора:

    docker stack deploy -c docker-compose.yml task_manager

Убедитесь, что сервис работает:

    docker service ls

Для тестирования сервиса: 

- Находим работающий контейнер базы данных или сервиса:
    ```
    docker ps
    ```
- Скопируйте `container id` и безжалостно убивайте:
    ```
    docker kill <сontainer id>
    ```

**ВАЖНО**: Восстановление базы данных может занять до 15 секунд. 

- Cмотрим обратно на наши контейнеры и убеждаемся, что наша база данных изменила свой `container id`

Остановка и очистка:

    docker stack rm task_manager

(Опционально) Если вы хотите полностью выключить режим Swarm на вашем компьютере:

    docker swarm leave --force
